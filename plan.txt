План разработки финансового трекера на основе созданной архитектуры. Каждый этап включает необходимые знания и навыки.

## Этап 1: Настройка зависимостей и базовой структуры
**Что нужно знать:**
- Работа с npm/yarn для установки пакетов
- Основы React и Vite
- Понимание package.json

**Задачи:**
- Установить react-router-dom для роутинга
- Выбрать и установить библиотеку для графиков (Recharts или Chart.js)
- Установить дополнительные зависимости для форм (Formik или react-hook-form)
- Проверить работу проекта командой `npm run dev`

## Этап 2: Реализация контекста аутентификации (AuthContext.jsx)
**Что нужно знать:**
- React Context API для управления глобальным состоянием
- Основы useState и useContext хуков
- Паттерны аутентификации в React

**Задачи:**
- Создать AuthProvider компонент
- Реализовать функции login, register, logout
- Добавить логику проверки токена/сессии
- Интегрировать localStorage для сохранения состояния

## Этап 3: Создание компонента авторизации (Auth.jsx)
**Что нужно знать:**
- Управление формой в React (controlled components)
- Валидация форм
- Обработка ошибок и загрузки
- CSS для стилизации форм

**Задачи:**
- Создать формы для входа и регистрации
- Добавить переключение между режимами
- Интегрировать с AuthContext
- Добавить валидацию полей (email, пароль)

## Этап 4: Настройка роутинга (App.jsx)
**Что нужно знать:**
- React Router v6 (BrowserRouter, Routes, Route, Navigate)
- Защищенные маршруты (PrivateRoute)
- Условный рендеринг на основе состояния аутентификации

**Задачи:**
- Обернуть приложение в BrowserRouter
- Настроить маршруты /login и /dashboard
- Создать защиту маршрутов для аутентифицированных пользователей
- Добавить редиректы после входа/выхода

## Этап 5: Создание страниц (LoginPage.jsx, DashboardPage.jsx)
**Что нужно знать:**
- Композиция компонентов в React
- Использование useNavigate для навигации
- Разделение логики между компонентами и страницами

**Задачи:**
- LoginPage: отобразить Auth компонент, добавить редирект
- DashboardPage: скомпоновать Dashboard и Menu компоненты
- Добавить layout компоненты если нужно

**Что нужно знать:**
- Работа с данными (массивы объектов для расходов)
- Библиотека графиков (например, Recharts: LineChart, PieChart)

## Этап 7: Реализация нижней навигации (Menu.jsx)
